如何找到设备对应的驱动(有没有加载)

如何查看一个设备对应哪个驱动？（如PCIe SD卡控制器，或者USB设备）

使用 lspci 或 lsusb 查看设备信息

Kernel driver in use即该设备所match的驱动；Kernel modules即驱动编译出的模块名。命名不一样的原因：

驱动名和模块名的命名不一致原因

  1. 模块名是加载模块时使用的名称（例如，使用 modprobe 或 insmod 时用的名字），通常是下划线分隔。
  2. 驱动名是设备驱动注册到内核时使用的名称（设备与驱动匹配时），可能使用短横线分隔。

主要为了保持向后兼容：早期文件系统对下划线和短横线的限制不同，在驱动注册后内核将短横替换为下划线并建立命名对应关系，所以加载驱动模块用的下划线名，而文件系统和驱动编译系统都用的短横名。
