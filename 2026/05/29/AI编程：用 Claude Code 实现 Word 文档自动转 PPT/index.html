AI 编程：用 Claude Code 实现 Word 文档自动转 PPT

本文记录了使用 Claude Code Skills 将 .docx 文档自动转化为 .pptx 演示文稿的完整过程，涵盖原理解析、工具链选型、工作流设计、样式提取与最终规则沉淀。

一、为什么要做 DOCX 转 PPTX？

工程师写技术报告习惯用 Word，但汇报时需要 PPT。手动搬运内容费时费力：调格式、贴截图、对齐样式，一份 10 页的报告往往要花 30 分钟以上做 PPT。

核心诉求很简单：

  • 内容从 Word 来：标题、正文、图片、表格
  • 样式从模板来：已有的公司 PPT 模板（颜色、字体、布局）
  • 输出一份新 PPT：内容 + 样式合并，开箱即用

这个任务非常适合 AI 编程：结构化输入、明确的转换规则、可验证的输出。

二、技术选型与工具链

2.1 为什么不用 python-pptx？

python-pptx 是 Python 生态中最常见的 PPT 生成库，但它有几个痛点：

维度 python-pptx pptxgenjs
模板支持 需要操作底层 XML Slide Master API 直观
图表能力 基础 丰富（Bar/Line/Pie 等）
样式控制 繁琐的 Emu 单位 英寸为单位，直觉友好
AI 生成友好度 一般 高（JSON-like 配置）

最终选择 pptxgenjs（Node.js），配合 Claude Code 的 Skills 系统完成整个流程。

整个工作流依赖两个 Claude Code Skills：

Skills 是 Claude Code 的能力扩展机制，每个 Skill 包含工具脚本和使用指南，让 AI 具备特定领域的专业能力。

3.2 五步工作流

步骤 工具 命令 产出
1. 提取文本 pandoc pandoc input.docx -o output.md Markdown 格式的文档结构
2. 提取图片 unpack.py python scripts/office/unpack.py input.docx unpacked/ word/media/ 下的所有图片
3. 分析模板 unpack.py python scripts/office/unpack.py template.pptx unpacked/ theme XML + slideMaster XML
4. 生成 PPT pptxgenjs node create_pptx.js 最终 .pptx 文件
5. 验证输出 markitdown python -m markitdown output.pptx 文本内容校验

八、总结

8.1 技术栈总结

层级 技术 作用
内容提取 pandoc + unpack.py 从 .docx 获取文本和图片
样式分析 unpack.py + XML 解析 从模板 .pptx 提取颜色/字体/布局
PPT 生成 pptxgenjs (Node.js) 程序化创建幻灯片
内容验证 markitdown 验证输出完整性
流程编排 Claude Code Skills AI 驱动的自动化工作流

8.2 适用场景

  • 技术报告转汇报 PPT：工程师写 Word，自动生成演示文稿
  • 批量文档转换：多份报告使用同一模板批量生成
  • 模板标准化：确保所有输出 PPT 符合公司视觉规范

8.3 局限与改进方向

  1. 复杂排版：Word 中的复杂表格、嵌套列表目前需要人工调整映射规则
  2. 图片尺寸：当前使用固定尺寸，未来可根据图片原始比例自动计算
  3. 模板背景图：pptxgenjs 无法直接复用模板的背景图片，需要手动提取后作为 base64 嵌入
  4. 动画效果：pptxgenjs 不支持 PPT 动画，如需动画需后期手动添加
